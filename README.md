# Service Bus Explorer — Power Apps Code App

A React + TypeScript + Vite app deployed as a [Power Apps Code App](https://learn.microsoft.com/power-apps/maker/canvas-apps/code-apps-overview), providing a web UI to interact with Azure Service Bus queues.

## Features

- **Send messages** to a queue (auto base64-encoded)
- **Peek messages** with peek-lock from Main or Dead Letter queues via segmented toggle
- **Complete** messages (permanently removes from queue) with confirmation dialog
- **Dead-letter** messages (moves to Dead Letter sub-queue) with confirmation dialog
- **Abandon** messages (releases lock, returns to queue)
- **Auto-refresh** — re-peeks the active queue after sending a message
- **Per-message loading** — action target fades while the operation runs
- **Metadata pills** — sequence number, correlation ID, and scheduled enqueue time shown as tags
- **Base64 decoder** utility panel
- **Dark glass-panel UI** with colour-coded status bar

## Prerequisites

- Node.js 18+
- npm
- [Power Platform CLI (`pac`)](https://learn.microsoft.com/power-platform/developer/cli/introduction)
- A Power Platform environment with Code Apps enabled
- An Azure Service Bus namespace with Entra ID authentication enabled

## Setup

1. **Clone and install**
   ```bash
   git clone https://github.com/Griffy42/CodeApps-Power.git
   cd CodeApps-Power
   npm install
   ```

2. **Authenticate with Power Platform**
   ```bash
   pac auth create
   pac env select --environment "<your-environment-id>"
   ```

3. **Initialise the Code App** (creates `power.config.json`)
   ```bash
   npx power-apps init
   ```

4. **Create a Service Bus connection** in [make.powerapps.com](https://make.powerapps.com):
   - Go to **Connections** → **+ New connection** → **Service Bus** → **Microsoft Entra ID** auth
   - Enter your namespace endpoint (e.g. `sb://yournamespace.servicebus.windows.net`)

5. **Link the data source**
   ```bash
   npx power-apps add-data-source --api-id shared_servicebus
   ```
   - Select **No** for connection reference
   - Paste the connection ID from the connection URL in make.powerapps.com

6. **Configure the queue name** in `src/App.tsx`:
   ```ts
   const QUEUE_NAME = 'your-queue-name';
   ```

7. **Build and deploy**
   ```bash
   npm run build
   npx power-apps push
   ```

## Dev Commands

| Command | Description |
|---------|-------------|
| `npm install` | Install dependencies |
| `npm run dev` | Run locally with Vite dev server |
| `npm run build` | Production build |
| `npm run lint` | Lint with ESLint |
| `npx power-apps run` | Run locally with Power Apps context |
| `npx power-apps push` | Deploy to Power Platform |

## RBAC

The signed-in user needs **Azure Service Bus Data Owner** (or Data Sender + Data Receiver) role on the Service Bus namespace for the Entra ID connector to work.

## Project Structure

```
src/
  App.tsx          — Main component (send, peek, complete, dead-letter, abandon,
                     confirmation dialogs, per-message loading, auto-refresh)
  App.css          — Dark glass-panel theme, queue toggle, coloured action buttons
  index.css        — Global base styles (dark gradient background)
  main.tsx         — React entry point
  generated/       — Auto-generated by Power Apps CLI (do not edit)
    services/ServiceBusService.ts  — Service Bus connector SDK
    models/ServiceBusModel.ts      — TypeScript types
```

## Notes

- `power.config.json` is gitignored — it contains environment-specific IDs. Each developer creates their own via `npx power-apps init`.
- The `src/generated/` folder is auto-generated by the Power Apps CLI when you run `add-data-source`. Do not edit manually.
- SAS key authentication may be disabled by Azure Policy on some namespaces — use Entra ID auth in that case.
